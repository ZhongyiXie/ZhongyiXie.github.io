📖 题目描述

一个基于 SM3 哈希的 Token 验证系统：

  • 密钥 hmac_key 被强制控制为 32 字节（不足则补 \x00，超过则截断）
  • Counter 为 4 字节整数
  • Token = SM3(hmac_key || counter)
  • 输入总长 = 32 + 4 = 36 字节 < 64 字节（SM3 分组大小）
  • → SM3 只计算了 一个分组！

你需要：构造新的 counter 和对应 token，使得 counter 低 32 位 > 原 counter，且 token 正确。
不能直接知道 hmac_key，但可以利用 SM3 的 Merkle-Damgård 结构进行哈希长度扩展攻击。

📦 已知数据
SM3 哈希结果 (hex, 64 字符 = 256 bit)
Counter 值 (hex, 8 字符 = 4 字节)
HMAC_KEY 长度
32 字节 (未知内容)
SM3 分组大小
64 字节 (512 bit)
💡 提示
Hint 1 — SM3 填充规则
SM3 对 36 字节消息的填充：消息(36B) + "1"(1bit) + "00...0"(填充到448bit) + 长度(64bit=288)。
填充后整个分组 512 bit = 64 字节。我们拿到的哈希值就是这个分组压缩后的输出。
Hint 2 — 构造新 counter
新 counter = 原counter(4B) + SM3填充剩余部分 + 额外数据(如 \xff\xff\xff\xff)
这样 SM3(KEY || 新counter) 会被分成 2 个分组：
分组1 = KEY || counter || padding（与原来相同 → 哈希值 = 已知值）
分组2 = 额外数据 || 新padding → 以已知哈希值为 IV 继续计算
Hint 3 — 不使用 KEY 计算新哈希
SM3 的压缩函数 CF 是公开的。以已知的哈希值为初始向量 IV，对第二个分组（额外数据+padding）调用 CF，即可得到 SM3(KEY || 新counter)。
完整 padded 消息结构的 hex 就是新 counter。

📖 题目描述

SM4 加密系统使用 LCG（线性同余生成器） 生成密钥和 IV：

🐍 Python
MULTIPLIER = 6364136223846793005  # 乘数 a
ADDEND = 1                        # 加数 b
MASK = 0xff_ffff                  # 模数 2^24
ROUNDS = 5                        # 每次 5 轮迭代

def genRandom():
    global seed
    for _ in range(ROUNDS):
        seed = (seed * MULTIPLIER + ADDEND) & MASK
    return (seed >> 12) & 0xfff    # ⚠️ 只返回高 12 位！

你已知 IV = 连续 4 次 genRandom() 输出（每次 12 bit）。需要恢复 key = 后续 4 次输出 来解密数据。

⚠️ 参数已缩小：24-bit 模数，5 轮迭代，泄露 12 bit。低 12 位暴力搜索仅 4096 种可能。

📦 已知数据
IV (hex) — 4 次 genRandom 输出，每次 12 bit = 3 hex 字符
LCG 参数
a = 6364136223846793005, b = 1, MASK = 0xffffff, ROUNDS = 5
加密数据 (密文 hex)
💡 提示
Hint 1
每次 genRandom() 做 5 次 LCG → 返回高 12 位。IV[0] 的高 12 位已知，低 12 位未知(4096 种)。
Hint 2
暴力搜索低 12 位：对每种可能，迭代 5 轮得到下一状态，检查高 12 位是否匹配 IV[1]。匹配则继续验证 IV[2], IV[3]。
Hint 3
找到正确状态后，继续迭代 4 次 genRandom() 得到 key。用 IV 和 key 解密：每个明文字节 = 密文字节 XOR key字节[i%8] XOR iv字节[i%16]。
